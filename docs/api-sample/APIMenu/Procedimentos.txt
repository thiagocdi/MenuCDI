* pesquisar no windows, "Editar as Variaveis de Ambiente do Sistema"
	- clicar em "Variaveis de Ambiente..."
	- em "Variaveis do Sistema", clicar em "Novo..."

	- POR PADRAO, QUANDO INSTALADO NO WINDOWS, IRÁ MONTAR A CONEXAO COM O BANCO DE DADOS CONFORME O REGISTRO HKEY_CURRENT_USER\Software\CDI\Contas a Receber\Banco de Dados\
		* se nao for ambiente windows, ou nao houver esse registro, irá buscar conforme a variavel de ambiente "CDI_SQL_CONNECTION_STRING"
			- criar uma variavel de ambiente chamada "CDI_SQL_CONNECTION_STRING"
				- atribuir a string de conexao do banco de dados SQL Server a variavel
				- exemplo:
				Data Source=cdi-info-003;Initial Catalog=Barbi;Persist Security Info=False;User ID=sa;Password=Teste#123;Encrypt=False;

* Tabela PARAMETROS, preencher campos:
	CaminhoNuvem = 'http://145.223.26.230/c26ffd1b88a46962'			//CONFORME CLIENTE
	SenhaCaminhoNuvem = 'Barbi#123'									//CONFORME CLIENTE
	CaminhoNuvemGenerico = 'http://145.223.26.230/cdi4cbbf9c648c05' //igual para todos clientes
	SenhaCaminhoNuvemGenerico = 'CdiInfo#123'						//igual para todos clientes
    CK_EXEC_NUVEM = 1												// 1 = busca exec da nuvem, 0 = busca exec local

* abrir o prompt de comando do windows como administrador
setx ASPNETCORE_URLS "http://0.0.0.0:8000"

* instalar servico:
sc.exe create ApiMenu binPath= "C:\Exec\APIMenu\ApiMenu.exe" start=delayed-auto displayname= "CDI API Menu"

* iniciar servico:
sc start ApiMenu

* verificar status do servico:
sc query ApiMenu

* parar servico:
sc stop ApiMenu

* apagar servico:
sc delete ApiMenu
